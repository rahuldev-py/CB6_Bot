## Feature Map — 20 Required Features Before Public SaaS

### GROUP 1: Authentication & Billing (Foundation)

---

#### Feature 1: User Login System
**Priority:** Critical  
**Stack:** FastAPI + PostgreSQL + JWT  
**Requirements:**
- Email + password registration
- Google OAuth option
- Email verification on signup
- Password reset flow
- Session management (30-day tokens)
- Admin can impersonate users for support

**ETA:** Week 1 of infra sprint

---

#### Feature 2: Subscription Billing
**Priority:** Critical  
**Stack:** Razorpay (India) + Stripe (International)  
**Requirements:**
- Monthly + annual plan support
- Auto-renewal with email reminder 3 days before
- Failed payment retry (3 attempts, then suspend)
- Proration on plan upgrades
- Invoice PDF generation
- GST-compliant invoicing for Indian users
- Webhook handler for payment events
- Refund processing (manual, admin-triggered)

**Pricing to implement:**
- Starter: ₹999/month, ₹8,999/year
- Pro: ₹2,999/month, ₹27,999/year
- Elite: ₹7,999/month, ₹74,999/year
- Elite Pro-Firm: ₹14,999/month
- International: $29 / $79 / $199 / $499 / month

---

#### Feature 3: User Dashboard
**Priority:** Critical  
**Stack:** React/Next.js + FastAPI backend  
**Sections:**
- Home: Today's market label + active signals + risk score
- Signals: Setup alerts (grade, entry, SL, target, expiry)
- Journal: Trade log with R-multiple, setup tag, outcome
- Risk: Daily loss tracker, drawdown gauge, position size calculator
- Reports: Weekly PDF downloads, monthly analytics
- Account: Profile, subscription, billing, API keys
- ML Memory (Elite only): Model confidence dashboard

### GROUP 2: Signal & Trading Features

---

#### Feature 4: Paper/Live Mode Separation
**Priority:** Critical  
**Requirements:**
- User selects mode at account setup: Paper or Observe
- Paper mode: signals with hypothetical P&L tracking (no real orders)
- Observe mode: user reviews signals, logs own trades manually
- Live broker integration: Phase 2 (after 500 users, with full legal review)
- Mode displayed prominently on all dashboards (never ambiguous)
- No auto-execution in SaaS v1 — signals only

---

#### Feature 5: Broker Integration Safety
**Priority:** High (Phase 2)  
**Requirements:**
- Read-only broker connection for portfolio display
- Signal → user's terminal execution flow (copy-paste format)
- No direct order placement in v1 SaaS
- Broker API keys stored encrypted (AES-256)
- Clear disclaimer: "CB6 does not place orders. You place orders."
- Supported later: Fyers, Zerodha, Angel One, MT5

---

#### Feature 6: Trade Journal
**Priority:** Critical  
**Requirements:**
- Manual trade entry (symbol, direction, entry, SL, target, outcome)
- Auto-tagging with CB6 setup label (if signal was generated by CB6)
- R-multiple calculator
- Win rate by setup type
- Win rate by session (morning / afternoon / end-of-day)
- Win rate by market condition (trending / ranging / high-vol)
- CSV + PDF export
- Shared journal option (for community leaderboard, opt-in)

---

#### Feature 7: Setup Grading
**Priority:** Critical  
**Requirements:**
- Score 1–20 system (current internal scoring)
- Grade labels: A+ (≥17), A (≥14), B (≥10), C (< 10)
- Grade displayed on each signal alert
- Grade breakdown visible (which criteria passed/failed)
- Historical grade accuracy tracking (A+ signals win rate vs. B signals)

---

#### Feature 8: Risk Score
**Priority:** Critical  
**Requirements:**
- Per-trade risk score (0–100): accounts for market vol, news risk, H4 alignment, setup grade
- Portfolio risk score: aggregate exposure, correlation, drawdown pace
- Daily risk budget tracker (shows % of daily limit used)
- Alerts at 50%, 75%, 90% of daily risk limit
- Risk score history chart

### GROUP 3: Intelligence Features

---

#### Feature 9: Market Condition Label
**Priority:** High  
**Labels:**

#### Feature 10: ML Memory Report
**Priority:** High (Elite only)  
**Requirements:**
- Show DNN, CNN, RNN model prediction for each signal
- Confidence score (0–100%)
- Prediction history for last 30 trades
- Model accuracy tracking (auto-updated after each trade)

- Shadow mode label on all ML outputs (never presented as trade advice)

---

#### Feature 11: Telegram/WhatsApp Alert System

**Requirements:**

### GROUP 5: Scale Features

---

#### Feature 17: Usage Analytics
**Priority:** Medium  
**Track:**
- DAU/MAU ratio (engagement health)
- Signal open rate (% of alerts clicked)
- Journal entry rate (% of signal users who log trades)
- Feature adoption by plan tier
- Churn cohort analysis (which features prevent churn?)
- Upgrade funnel (Starter → Pro → Elite conversion rates)

**Stack:** PostHog (open-source, self-hosted) or Mixpanel

---

#### Feature 18: Server Monitoring
**Priority:** High  
**Requirements:**
- Uptime monitoring (99.9% SLA target)
- Alert on signal delivery failure > 2 min
- ML worker health check (auto-restart if stuck)
- Database query performance alerts
- Disk/memory/CPU alerts
- Telegram alert to admin on any critical failure

**Stack:** Grafana + Prometheus (or Sentry + UptimeRobot for simpler start)

---

#### Feature 19: Load Scaling Readiness
**Priority:** Medium (required before 500 users)  
**Requirements:**
- Stateless API design (horizontal scaling possible)
- Database connection pooling (PgBouncer)
- Redis caching for signal + report data (reduce DB load)
- Celery/RQ task queue for signal generation (async, not blocking)
- CDN for static assets (Cloudflare)
- Staging environment that mirrors production

---

#### Feature 20: Support Ticket System
**Priority:** High  
**Requirements:**
- In-app support form (email + ticket ID)
- Telegram @CB6Support bot for priority users
- Response SLA: Elite < 4h, Pro < 12h, Starter < 48h
- Ticket categories: billing, signal, technical, account
- Knowledge base / FAQ (reduces ticket volume)

**Stack:** Freshdesk free tier or Crisp chat (both have free plans)

## Technology Stack Decision

| Layer | Technology | Reason |
|---|---|---|
| Backend API | FastAPI (Python) | Existing CB6 codebase is Python — minimal context switch |
| Frontend | Next.js + TailwindCSS | React ecosystem, SSR for SEO, fast |
| Database | PostgreSQL | Production-grade, SQL, JSONB support |
| Cache | Redis | Signal caching, session storage |
| Task Queue | Celery + Redis | Async signal generation, ML jobs |
| Auth | JWT + FastAPI-Users | Standard, well-documented |
| Billing | Razorpay (India) + Stripe (Intl) | Best coverage for both markets |
| Hosting | DigitalOcean / Hetzner (start) → AWS (scale) | Cost-effective start, scale later |
| Monitoring | Grafana + Prometheus | Open-source, free |
| Analytics | PostHog (self-hosted) | Free, privacy-respecting, powerful |
| CDN | Cloudflare | Free tier covers 500 users easily |
