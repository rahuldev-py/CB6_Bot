# Client Intake Form — Rahul (rahuldev-py)

> Ask these questions BEFORE quoting a price or starting work.
> Send this as a message after a client shows interest. Copy-paste the relevant section.

---

## Quick Message to Send First

```
Thanks for reaching out. Before I give you an accurate quote, I have a few quick questions.
This helps me scope the work correctly so there are no surprises later.
```

---

## Section A — Project Goal (ask always)

1. **What is the end goal of this project?**
   *(What should the system DO when it's working correctly?)*

2. **What is currently happening vs what should be happening?**
   *(If it's a bug fix: describe the wrong behaviour. If it's a new build: describe the desired outcome.)*

3. **How will you measure success?**
   *(What does "done" look like to you?)*

---

## Section B — Current Setup (ask always)

4. **What do you already have?**
   - [ ] Existing code (share repo or zip)
   - [ ] Working script with one bug
   - [ ] Nothing yet — starting from scratch
   - [ ] Someone else's code I need to understand/fix

5. **What is your tech stack?**
   *(Python version, libraries, OS, VPS/local machine)*

6. **What broker or platform are you using?**
   *(e.g. Fyers, Zerodha, MT5/GFT, Binance, Interactive Brokers, TradingView)*

7. **Do you have API access already?**
   - [ ] Yes — API key/credentials ready
   - [ ] Yes — but it's broken or expired
   - [ ] No — need to set it up

---

## Section C — Technical Details (ask for bugs/integrations)

8. **What is the exact error message or wrong behaviour?**
   *(Paste the full error traceback if there is one)*

9. **When does it break?**
   *(At startup? During market hours? Only on first trade? Randomly?)*

10. **What did you change before it stopped working?**
    *(New library? New broker? Moved to VPS? Upgraded Python?)*

11. **Do you have log files?**
    *(Share the relevant section — last 50 lines before the error)*

---

## Section D — Project Constraints

12. **What is your deadline?**
    - [ ] Flexible (no rush)
    - [ ] Within 1 week
    - [ ] Within 48 hours (emergency rate applies)
    - [ ] Today (emergency rate applies)

13. **What is your budget range?**
    *(Being specific helps me scope correctly. I won't quote above your budget without telling you first.)*
    - [ ] Under $100
    - [ ] $100–$300
    - [ ] $300–$800
    - [ ] $800+

14. **Are there any security restrictions?**
    *(Can you share the code? Are there NDAs? Can I access your VPS remotely?)*

---

## Section E — Deliverable Definition

15. **What exactly should I hand over at the end?**
    *(Running script? Full codebase? Documentation? Video walkthrough?)*

16. **Who will maintain the code after delivery?**
    *(You? Your developer? Do you need it to be well-documented?)*

17. **Are there existing coding standards or naming conventions I should follow?**

---

## Section F — Payment

18. **How do you prefer to pay?**
    - [ ] Upwork escrow (recommended — both parties protected)
    - [ ] Wise direct transfer
    - [ ] PayPal

19. **Are you okay with milestone payments for larger projects?**
    *(Projects over $300: 50% upfront, 50% on delivery)*

---

## Red Flags — Decline if any of these apply

- Client cannot describe what "done" looks like
- Client wants ongoing strategy optimisation for a fixed price
- Client asks for guaranteed trading profits
- Client wants access to their clients' broker accounts
- Client cannot share any code or context ("just fix it blindly")
- Budget is under $50 for work that needs more than 2 hours
- Client is in a country with payment issues and insists on unusual payment methods
- Client wants you to scrape a site that explicitly prohibits it in ToS

---

## After You Get the Answers

1. Review answers — does the scope match your skills and timeline?
2. Write a clear scope statement: "I will deliver X. I will not deliver Y."
3. Quote a fixed price based on scope (not hours)
4. Send the quote with a 48-hour validity
5. Get 50% upfront before writing a single line of code
